# Roadmap

## Current focus

Henka Engine is currently focused on a clean visible 3D foundation:

- engine lifecycle
- logging
- memory wrappers
- math and timing
- platform abstraction
- input handling
- renderer abstraction
- mesh and shader-backed draw path
- texture and material foundation
- asset manager foundation
- local persistence foundation
- lightweight in-window UI overlay
- early OBJ model loading
- minimal scene and camera flow
- sandbox startup and movement

## Near-term roadmap

The next likely layers are:

- safer camera controls
- broader model loading beyond the current OBJ subset
- richer material support
- stronger asset management
- stronger persistence and save policies for external game projects
- clearer external project integration beyond the current starter template
- scene growth beyond the current lightweight entity model
- broader OBJ and model import coverage after the current foundation has settled
- richer in-window inspection and QA tools beyond the current lightweight sandbox panels and object inspection
- stronger external-project packaging and validation beyond the current starter template and validation script

## 2.5D direction

Henka Engine is planned to support 2.5D as a distinct workflow rather than treating it as an afterthought. That later work should include:

- camera constraints
- layered depth
- parallax
- side-scroller support
- top-down support
- isometric support

## Longer-term work

Longer-term milestones can expand into:

- full 2D renderer
- audio
- scripting
- physics
- editor workflows
- other renderer backends
